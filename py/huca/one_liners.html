HUCA One-Liners

The scene structure:

  1. Scene: what we conceive at a given time, a conception of a state, action or movement. It has a specified ground, and usually a specified time and location.
  2. Verbial: the main relation in the scene. The state, activity or movement, by virtue of which it is a scene.
  3. Argument: an entity participating in the scene (including location).
  4. Adverbial: a secondary relation within the scene. does not correspond to an object, but to a relation (including temporal relations).

Internal structure of complex units (i.e., units with sub-units):

  1. Center & Elaborator: if the complex unit contains one or more relations applied to a single argument, the relations are marked as Elaborator and the joint argument as a Center. If two units correspond to the same entity and it is hard to say which elaborates which, both are Centers.
  2. Center & Link: if the entity in question contains a relation between two or more entities (usually coordination), we mark the relation as a Link and the related units as Centers.

Linkage (relation between scenes):

  1. Argument Scenes: serve as an argument in another scene. Should be marked as any Argument.
  2. Elaborator Scenes: elaborate on some unit. should be annotated as any Elaborator.
  3. Grounding Scenes: scenes that describe the ground of another scene.
  4. Linked Scenes (including Adverbial scenes): the default setting. The relation should be marked as Linker, and the linked scenes should be marked as Linked Units.

Note: even if not linked to other scenes, a scene should by default by annotated as a Linked Unit.
